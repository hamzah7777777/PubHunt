## Section Alternation Palette

Sections across the page MUST alternate background through these four brand colors in a playful, magazine-style rotation. Use this exact cycle to drive the rhythm:

1. **Section A** — `brand` background (#4503FF), `white` text
2. **Section B** — `secondary-brand` background (#9747FF), `white` text
3. **Section C** — `tertiary-brand` background (#C3FBDC), `heading` text
4. **Section D** — `quaternary-brand` background (#A9CDF1), `heading` text

Neutral sections (`neutral-primary-soft`) are allowed only as breathing space between vivid blocks. The opening hero, feature sections, and CTA blocks should each pick a different swatch from the cycle so consecutive sections never repeat the same color.

## Surface Cards on Colored Sections

When a card sits on top of a colored section, its background MUST be a lighter derivative (a `*-softer` or `*-soft` variant) of that section's color, never neutral white:

| Section background | Card background |
|---|---|
| brand | brand-softer |
| secondary-brand | secondary-brand-softer |
| tertiary-brand | tertiary-brand-softer |
| quaternary-brand | quaternary-brand-softer |
| neutral-primary-soft | neutral-primary or brand-softer |

This keeps the surface anchored in the same hue family as the section while still reading as elevated content.

## Semantic Usage Rules

- Page/section backgrounds: rotate through `brand`, `secondary-brand`, `tertiary-brand`, `quaternary-brand` per the alternation palette above; use `neutral-primary-soft` only sparingly as a reset
- Primary buttons: brand background with the multi-layer brand shadow (see `buttons.md`)
- Headings: heading text color (or white when sitting on `brand`/`secondary-brand` sections)
- Body text: body text color (or white-with-90%-opacity on dark vivid sections)
- CTA links: fg-brand text color, underlined
- Default borders: border-default (a hard, near-black line — this is the magazine-print baseline)
- Status borders match intent: success → border-success, danger → border-danger, warning → border-warning
- Disabled: disabled background + fg-disabled text

## Prohibited

- No raw hex/rgb values in component code — always use design tokens
- No brand text color for long-form paragraphs
- No accent text tokens (fg-purple, etc.) for body copy or navigation
- No flat all-neutral page composition — sections must rotate brand colors per the alternation palette
- No manual light/dark value swapping — let the design tokens handle it
